# Foundry SDK を使った Contoso Travel（プロンプトエージェント）の構築

ここまでに _インフラのセットアップ_ と _開発環境のセットアップ_ が完了し、Azure サブスクリプション内の既存の Foundry プロジェクトに接続された GitHub Codespaces が起動している状態であることを確認してください。

このラボパスでは、ノートブックを順番に実行します。Contoso Travel エージェントの構築から始まり、カスタムデータとツールによるグラウンディング、評価とトレーシング、そして攻撃への脆弱性を評価する AI レッドチームスキャンの実行までを体験します。

### 2.1 ノートブックの実行方法

GitHub Codespaces 環境には、Jupyter ノートブックをサポートする拡張機能とツールがあらかじめインストールされています。ノートブックを「実行」するには、以下の手順に従ってください:

1. Visual Studio Code でノートブック（`.ipynb` ファイル）を開く
1. **カーネルの選択** をクリックし、**Python 3.12** を選択する
1. まずすべての出力をクリアしてから、（「すべて実行」ではなく）**セルを1つずつ実行する**
1. 次に進む前に結果を確認し、各ステップの内容を理解する

**重要**: ラボが完了してコードの内容を十分に理解するまで、ノートブックの最終セルを実行しないでください。最終セルを実行すると、ノートブックの出力だけでなく、Foundry に作成されたエージェントなどの関連アセットもすべて削除されます。
